SSRS SDK for PHP User Guide

Introduction

SSRS SDK for PHP is designed to make it easier for PHP applications to use reports created with Microsoft SQL Server Reporting services (SSRS).

Requirements

This toolkit requires the PHP SOAP Extension to be installed and enabled.

Installation and Configuration

The assumption is that PHP is already installed and configured on the machine where the SSRS SDK for PHP is going to be installed.


The toolkit does not have any dependency on the host OS so it can run on Microsoft Windows, Linux, or Mac OSX machines.

  1. Create a folder ex, C:\PHPLib\ssrsphp and copy the entire content of the folder bin.

 

2. In the PHP.INI file:

a.       Add the path to the folder created in step 1 to the 'include_path' directive in php.ini. e.g.

include_path = ".;C:\PHPLib\ssrsphp"

 

b.       Enable php_soap.dll in php.ini: search for 'extension=php_soap.dll' and remove the semicolon (;) in front.

Using the toolkit

The toolkit is composed a library (SSRSReport.php) that needs to be included once in the application.

require_once 'SSRSReport.php';

 


The SSRSReport.php file contains definition of SSRSReport class. The SSRSReport class provides the methods to connect to the SQL Server Reporting Services Server.

 

The typical usage pattern of the library is:

 

1.       Instantiate the SSRSReport class and establish a connection to the SQL Server Reporting Service by providing the required credentials.

2.       Retrieve list of available reports [optional]

3.       Initiate a report (Consumption of a report requires reporting server to load that report and create a session for it).

4.       Set the value for the required parameters.

5.       Render the report in the required format.

Why the 2 postfix on some APIs?

 

The concept of PageCount mode was introduced in reporting service 2008 (RS2008), which was not supported in reporting service 2000 (RS2000).

 

For loading a report into execution, RS2000 exposes a method called LoadReport, which returns a structure of type ExecutionInfo. RS 2008 extended this ExecutionInfo structure by adding a member variable PageCountMode to hold Page count mode, the new structure is known as ExecutionInfo2.

 

For RS 2000 backward compatibility, RS 2008 kept the API names intact (LoadReport, Render, SetExecutionParameters, GetExecutionInfo) and exposed APIs names end with 2 (LoadReport2, Render2, SetExecutionParameters2, GetExecutionInfo2) which supports Page Count Mode. For the toolkit we decided to keep the same naming convention so it is easier to find documentation on the MSDN site.

Using the SSRSReport class

Connecting to SSRS (SSRSReport Constructor)

 

The first step is to instantiate the SSRSReport class and establish a connection to the SQL Server Reporting Services service. Credentials for a valid user on the server will have to be provided.

 

Syntax :

SSRSReport ($credentials, $url, $proxy)

 

Parameters

Description

credentials

Type: credentials.

Connection to Report Server Web Services requires user credentials (Please refer to either the document, .\Samples\HelloWorld\setup\Server_Setup.html or the Documentation available on MSDN for how to add a user (role assignment) in Report Server)

 

url

Type: string.

The Report server's web service URL. url can be found using Reporting Service Configuration Manager (RSCM). Open RSCM window from Start -> Programs -> Microsoft SQL Server 2008 -> Reporting Services Configuration Manager, click on Web Service URL in the Connect pane, you can see the URL in Report Server Web Service URLs section.

proxy

Type: proxy

If a proxy is used, then this argument is used to supply proxy information. The default value is null.

 

 

Exceptions

errorCode

errorMessage

null

Failed to connect to Reporting Service. Make sure that the url ([report server url]) and credentials are correct.

 

 

Example:

define("UID", "PHPDemoUser");

define("PASWD", "Passwrd");

define("SERVICE_URL", "http://localhost/ReportServer_SQLEXPRESS/");

try

{

$ssrs_report = new SSRSReport(new Credentials(UID, PASWD), SERVICE_URL);

}

catch (SSRSReportException $serviceException)

{

echo $serviceException->GetErrorMessage();

}

 

 

ListChildren

 

After a connection to the SSRS server has been establish, the listChildren method can be used to retrieve the list of children of a specified folder in the Report Server. More documentation on Report structure can be found here

 

Syntax :

CatalogItem[] ListChildren($Item, $Recursive)

 

Note: children can be Report, DataSource, LinkedReport, Resource, Folder or Model

Parameters

Description

Item

Type: string.

The full path name of the parent folder whose children we need to get. The default value /, indicates the root folder.

Recursive

Type: boolean.

A boolean value that indicates whether to return the entire tree of child items below the specified item. The default value is TRUE.

 

Return Type

Description

CatalogItem []

Collection of CatalogItem. Each CatalogItem represents an item in the report server database. Item can be Report, DataSource, Linked Report, Resource, Folder or Model

 

 

Exceptions

errorCode

errorMessage

rsInvalidItemPath

The path of the item [path] is not valid. The full path must be less than 260 characters long; other restrictions apply. If the report server is in native mode, the path must start with slash

 

The Following example shows how to get all Reports from Report Server:

define("UID", "PHPDemoUser");

define("PASWD", "Password");

define("SERVICE_URL", "http://localhost/ReportServer_SQLEXPRESS/");

$ssrs_report = new SSRSReport(new Credentials(UID, PASWD), SERVICE_URL);

$reports = array();

if ($catalogItem->Type == ItemTypeEnum::$Report) {

$reports[] = array(

"Name" => $catalogItem->Name,

"Path" => $catalogItem->Path

);

}

}

}

catch (SSRSReportException $serviceException)

{

echo $serviceExcprion->GetErrorMessage();

}

GetReportParameters

Once a report has been identified, for instance by using the listChildren method, GetReportParameters can be used to get the metadata for the parameters required by a report, including names, types, and valid values. The return value will be a collection of ReportParameter objects, i.e. for each parameter in the report there will a corresponding metadata.

Syntax :

ReportParameter[] GetReportParameters($Report, $HistoryID, $ForRendering,

$Values, $Credentials)

 

Note: While defining parameters during report design, user can specify valid values and default values. For example if Month is a parameter then user can define valid values as January to December and default value as June. The ReportParameter class includes this information,

More information on the ReportParameter properties is available on MSDN.

 

 

Prameters

Description

Report

Type: string.

The full path name of the report.

HistoryID

Type: string.

The ID of the report history snapshot. See additional information section for History ID details. The default value will be null

ForRendering

Type: Boolean

 

ForRendering

HistoryID

Description

False

Null/ Valid-HistoryID

When ForRendering is False, the value supplied for HistoryID is ignored.

The returned parameter meta data will not contain valid values if parameter is query based.

True

Valid-HistoryID

The parameter meta data returned represents the parameter data that was used when the report history snapshot was created.

True

null

 

If any parameters values are based on a query, then setting ForRendering to true cause the returned parameter meta data to contains valid values for these parameters.

 

Default value will be False

Values

Type: ParameterValue[]

The parameter values (ParameterValue objects) that can be validated against the parameters of a report that is managed by the report server. Additional details are available on MSDN

Default value will be False

Credentials

Type: DataSourceCredential[]

Credentials for the data sources used in the Report (DataSourceCredential[] objects). If values for any of the parameters are based on a query to a data source the DataSourceCredential list contains the credentials needed to access the corresponding databases . Additional details are available on MSDN.

Default value will be null

 

Return Type

Description

ReportParameter[]

Collection of ReportParameter. Each ReportParameter represents metadata of a parameter in the report. Additional details on the ReportParameter object can be found on MSDN.

LoadReport2

Before a report can be used, it needs to be initialized on the server by calling the LoadReport2 method. The method loads a report in the server and set it into a new execution context. it returns an ExecutionInfo2 object containing information for the loaded report.

 

Syntax :

ExecutionInfo2 LoadReport2($report, $historyID)

 

 

Note: Once user invokes LoadReport2, Report Server will load the report and creates a session for it (identified by a session id, also known as execution id). The ExecutionInfo2 object returned by LoadReport2 contains a member variable ExecutionID which holds session id. When user wants to perform any operations on the loaded report (ex: Setting execution parameters, rendering etc...), the SSRS_Report library will internally use the appropriate session id before calling the server.

This method should be called before calling the Render2 method.

 

Parameters

Description

report

Type: string.

The full path name of the report.

HistoryID

Type: string.

The ID of the report history snapshot. See additional information section for HistoryID details. The default value will be null.

 

Return Type

Description

ExecutionInfo2

An ExecutionInfo2 object containing information for the loaded report.

 

If user has specified a valid value for HistoryID then Report Server will load the report from cache, in this case ExecutionInfo2 will represent a report which is ready to be rendered.

 

If user has specified null as HistoryID then ExecutionInfo2 will represent a report that needs to be processed (processing report means executing reports data set queries) before rendering.

 

Exceptions

errorCode

errorMessage

Description/How to fix

rsItemNotFound

The item '[report path specified by user]' cannot be found

Report Server will throw this error if it failed to locate the report specified by user. Make sure the report path is correct.

 

The Following example shows how to Load a report into a new execution context:

define("UID", "PHPDemoUser");

define("PASWD", "Password");

define("SERVICE_URL", "http://localhost/ReportServer_SQLEXPRESS/");

define("REPORT", "/AdventureWorks 2008 Sample Reports/Employee Sales Summary 2008");

try

{

$ssrs_report = new SSRSReport(new Credentials(UID, PASWD), SERVICE_URL);

$executionInfo = $ssrs_report->LoadReport2(REPORT, NULL);

if$executionInfo->CredentialsRequired)

{

echo "The report data source ". $executionInfo->DataSourcePrompts[0]->Name . "Requires credentials";

}

else

{

echo "Report data source does not require credentials";

}

}

catch(SSRSReportException $serviceException)

{

echo $serviceExcprion->GetErrorMessage();

}

Out of the box SQL Server Reporting Services includes seven rendering extensions: HTML, Excel, Word, CSV or Text, XML, Image, and PDF. This API returns the list of extensions report server supports

Syntax :

Return Type

Description

A collection of Extension objects. Each Extension object represents an extension supported by Report Server.

Render2

This is the last step of the process where the report, in the required format, is delivered to the client, the API processes a specific report and renders it in the specified format.

Syntax:

byte[] Render2($renderType, $PaginationMode, &$Extension,

&$MimeType, &$Encoding, &$Warnings, &$StreamIds)

 

 

 

Parameters

Description

renderType

Type: RenderAs*.

 

renderType can be object of type RenderAsPDF, RenderAsHTML, RenderAsEXCEL, RenderAsIMAGE, RenderAsWORD, RenderAsMHTML, RenderAsCSV or RenderAsXML

 

Used to specify the format in which user want report server to return a report. For each format there will be a corresponding component in the Report Server which generate report in the specific format, these components are known as Rendering Extensions.

This argument is also used to specify additional settings specific to the rendering extension used.

 

You should be careful while specifying this argument because the supported list of extensions varies depending the versionof SQL Server used, for instance SQLEXPRESS Report Server supports 4 rendering extensions: PDF, HTML, EXCEL and IMAGE formats, while SQL Server support 8 extensions.

Suppose there is a report in report server with 50 pages, and you want only pages from 1 to 5 in PDF format, then you can use Render2 API as follows:

 

$PdfFormat = new RenderAsPDF(); //select PDF Rendering Extension

$PdfFormat->StartPage = 1; //setting specific to PDF Rendering Extension

$PdfFormat->EndPage = 5; //setting specific to PDF Rendering Extension

$reportAsPDF = $SSRS_Report->Render2($PdfFormat, PageCountModeEnum::$Estimate, $Extension, $MimeType, $Encoding, $Warnings, $StreamIds);

PaginationMode

Type: PageCountModeEnum.

 

Please refer Additional information section for more details about Page Count Mode.

Extension [out]

Type: string

The file extension corresponding to the output stream.

MimeType [out]

Type: string

The MIME type of the rendered report.

Encoding [out]

Type: string

The encoding used when report server renders the contents of the report.

Warnings [out]

Type: Warning[]

An array of Warning objects that describes any warnings that occurred during report processing.

StreamIds [out]

Type: string[]

The StreamIds is collection of ids that will identify external resources associated with the report (such as image resource i.e. the image that represents the pie chart, graphs in the report for example).

If user want to retrieve an external resource associated with a report, then he can use RenderStream API by passing StreamId corresponding to the resource.

 

Return Type

Description

Byte[]

A byte (char) array of the report in the specified format.

 

Exceptions

errorCode

errorMessage

Description/How to fix

rsReportParameterProcessingError

name of a parameter defined for this report (ex:EmployeeID)

This error means the parameter (i.e. value of errorMessage member variable) value will come from data source which requires credentials and you didnt provided the required credentials.

 

Use SetExecutionCredential2 API before invoking Render2.

define("PASWD", "Passw0rd!");

define("REPORT", "/AdventureWorks 2008 Sample Reports/Employee Sales Summary 2008");

{

SERVICE_URL);

$ ssrs_report->LoadReport2(REPORT, NULL);

$htmlFormat->StreamRoot = './images/';

$result_html = $ ssrs_report->Render2($htmlFormat,

PageCountModeEnum::$Estimate,

$Extension,

$MimeType,

$Encoding,

$Warnings,

$StreamIds);

echo '<div style="overflow:auto; width:1000px; height:700px">';

echo $result_html;

echo '</div>';

}

Catch(SSRSReportException $serviceExcprion)

{

echo $serviceExcprion->GetErrorMessage();

}

 

The output of this code will show the report in a frame. Note that you wont be able to see the images (resources) in this report; the sample code provided under the RenderStream API reference shows how to handle resources (images for example) associated with a report.

This report contains 3 images (external resources) so the array $StreamIds contains id of each image; we can use RenderStream API to retrieve these images by passing these ids. The html representation (in the variable $result_html) of the report will contain src tag for each image. We can instruct HTML rendering extension about how to construct the URLs to these images. Here we set value of RenderAsHTML::StreamRoot as './images/', this cause HTML rendering extension to create all urls with relative path ./images. This makes easier to save all the images on a predefined folder without the need of updating all the links in the report. For example

src=./images/ C_23iT0_1

Where C_23iT0_1 is stream id.

RenderStream

RenderStream returns an external resource that is associated with a rendered report. When you use Render2 method to render a report in HTML format, if the report contains images, then the returned html stream will not contain the images. In this case Render2 will return streamIds for the object like images, and you have to use RenderStream to retrieve these streams.

You will not face this problem for the other render types like PDF, EXCEL, DOCX or IMAGE, because that includes images in their stream.

Additional Information

What is Caching in Report Server?

Report Server administrator can decide if a report should be cached or not. Caching a long running report will allow the report to be rendered faster and give a better experience for the user.

If caching is enabled for a report, then report server can return the report from cache, when we request for a report stream using Render2, otherwise report server has to run the queries defined in the report before returning the stream.

Each cashed entry of a report is known as a snapshot. There are two main types of report caching:

         Execution Cashing: Execution snapshot will be kept in the cache only for a specified period of time.

         History Caching: History Snapshots that will store several cached versions of the report, each such snap shot is identified by a unique History ID, later using HistoryID we can retrieve the snapshots.

For more details about caching refer this URL.

What is PageCount Mode?

A Report can contain multiple pages. The ExecutionInfo2 structure contains a member variable NumPages which holds number of pages in a report. The second parameter of Render2 API instruct report server the way it should calculate the value of NumPages. You can call GetExecutionInfo2 API to get updated ExecutionInfo2 structure after invoking Render2. The possible values of second parameter of Render2 API are:

PageCountModeEnum::$Actual

Specifies that the report server should calculate page count value before the report is rendered. In this case you will get the total number of pages in the report from the return value of GetExecutionInfo2.

PageCountModeEnum::$Estimate

Specifies that the report server should calculate page count value as the report is viewed. In this case you will get number of pages as 2 (for the current page plus one additional page).

 

The following sample code assumes that Multiple_Page_Report is a report with 720 pages:

define("REPORT", "/Multiple_Page_Report");

$ssrs_report->LoadReport2(REPORT, NULL);

 

$htmlFormat = new RenderAsHTML();

$htmlFormat->Section = '1';

$result_html = $ssrs_report->Render2($htmlFormat,

PageCountModeEnum::$Actual,

$Extension,

$MimeType,

$Encoding,

$Warnings,

$StreamIds);

 

$exeInfo = $ssrs_report->GetExecutionInfo2();

Echo Number of Pages:. $exeInfo->NumPages;

Output:

Number of Pages: 720

 

In the sample code we used Render2 API to render the first page (we used the Section member variable of RenderAsHTML class to specify the page number) of the report in HTML format with PageCount mode as Actual. In this case even though we asked for the first page of the report, since we specified the value of PageCount mode as Actual, report server will process the entire report to find the total number of pages in the report.

 

define("REPORT", "/Multiple_Page_Report");

$ssrs_report->LoadReport2(REPORT, NULL);

 

$htmlFormat = new RenderAsHTML();

$htmlFormat->Section = '1';

$result_html = $ssrs_report->Render2($htmlFormat,

PageCountModeEnum::$Estimate,

$Extension,

$MimeType,

$Encoding,

$Warnings,

$StreamIds);

 

$exeInfo = $ssrs_report->GetExecutionInfo2();

Echo Number of Pages:. $exeInfo->NumPages;

Output:

Number of Pages: 2

 

This sample code is same as the previous one expect, here we specified PageCount mode as Estimate, so the SSRS server will not process the entire report to find the total number of pages.

 

Use Actual if you want to calculate page count in advance before the report is displayed. Note that if you set PageCountMode to Actual, the entire report must be processed to get a valid page count, increasing wait time before the report is displayed.

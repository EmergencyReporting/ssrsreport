Overview

This document provides setup documentation, step-by-step instructions, and a written script for showing the PHP sample application which uses SSRSReport Library to consume SQL Server Reporting Service.

This demo is aimed to show the reporting capabilities of SQL Server 2008 and the usage of SSRSReport Library to consume those capabilities.

Key Messages

1.       You will learn how to open, create and deploy SQL Server Reporting Services reports using Report Designer.

2.       You will learn how to consume embedded SQL Server Reporting Services reports using URL Access.

3.       You will learn how to consume SQL Server Reporting Services reports using SSRSReport Library.

2. Click on the sort button near the Store and Sale Amount header and show the sort in action.

         So we have created and deployed TopStoresBegin Report.

         In this demo we will show you how to use SSRSReport toolkit (which internally uses Report Server Web service) to consume the above report.

         The Report Server Web service end point allows accessing of the complete functionality of Reporting Services.

         When first opened, the page displays report in HTML format.

3. Open the sample application ConsumeReportingService.php file located under the HelloWorld\Code folder.

 

         Now we'll explain how to use the SSRSReport library to load a report, set execution parameters and render and Sort.

         The SSRSReport constructor receives three parameters, credentials, Report Server Web Service URL and proxy settings (which is optional).

         The SSRSReport library uses SOAP over HTTP to access Report Server Web service.

         The SSRSReport class creates two instance of the SoapClient class in order to connect with the Report Server Web Service.

         Report Server exposes two Web Services.

◦       ReportService2005.

◦       ReportExecution2005.

         The ReportService2005 endpoint contains methods for managing objects in a Report Server. The WSDL for this endpoint is accessed through ReportService2005.asmx?wsdl.

         The ReportExecution2005 endpoint contains methods to customize report processing and rendering from a report server. The WSDL for this endpoint is accessed through ReportExecution2005.asmx?wsdl.

         SoapClient constructor has the following parameters:

◦       wsdl: URI of the WSDL file.

◦       options: An array of options. For HTTP authentication, the login and password options can be used to supply credentials, for Proxy settings proxy_host, proxy_port, proxy_login and proxy_password can be used.

 

 

4. Show the LoadReport2 method at line 56 of ConsumeReportingService.php file.

         Now, having an instance of the SSRSReport class, we can make calls to any methods from this class which internally uses SOAP API to access the reporting service Web Service.

         To load a report from the report server database we are going to use SSRSReport::LoadReport2 method.

         SSRSReport::LoadReport2 method has the following parameters:

◦       report: The full name of the report.

◦       historyID: The history ID of the snapshot, the default will be NULL.

In this sample application REPORT (first parameter of LoadReport2 API) is a constant, which has the value: /AdventureWorks 2008 Sample Reports/TopStoresBegin

         The SSRSReport library uses the LoadReport2 web service method to load the specified report, parse the return value of this web service (execution information), build a structure from this return value and return it to client application.

         The LoadReport2 web service will create a new session for the loaded report (identified by a session id), we can see this session id from the return value of SSRSReport::LoadReport2.

$executionInfo-> ExecutionID

         Rest of the web service methods, which performs operations on top of this loaded report, requires this exection id (session id) as SOAP header, SSRSReport will take care of this.

 

 

 

 

5. Show the method SetExecutionParameter2 method at line 69 of ConsumeReportingService.php file

         For viewing report using Report Manager, we have specified the parameters; we will use SSRSReport::SetExecutionParameter2 methods to specify parameters.

         SSRSReport::SetExecutionParameter2 method has the following parameters:

◦       parameters: An array of ParameterValue objects. ParameterValue is a class to hold information about a parameter.

◦       parameterLanguage: locale identifier, the default will be en-us.

         The SSRSReport library uses the SetExecutionParameter2 web service method which sets and validates parameter values associated with the current report execution, the SSRSReport library will parse the return value of this web service (execution information), build a structure from this return value and return it to client application.

 

 

 

 

 

6. Show the method Render2 at

ConsumeReportingService.php file

         We will use SSRSReport::Render2 to get the report stream in a specified format.

         SSRSReport:: Render2 method has the following parameters:

◦       renderType: An object of type RenderAsCSV, RenderAsEXCEL, RenderAsHTML, RenderAsIMAGE, RenderAsMHTML, RenderAsPDF, RenderAsWORD, RenderAsXML

By default all the active links (links to other reports, toggle, sort links etc...) in a report are post back call to report server. Using RenderAsHTML::ReplacementRoot option we can redirect all such call to our php file. The function getPageURL() retruns the url of current page which is: http://localhost:50002/ConsumeReportingService.php.

If the report contains any resources such as images (our report contains an image representing pie chart), then we can instruct the report server about how to construct the URLs to these resources. Here we will use RenderAsHTML::StreamRoot member to instruct report server for creating all urls with relative path ./images

◦       paginationMode: The mode by which the report is processed. Possible values are:

◦       Extension: [out] the file extension corresponding to the return value. For example if we pass renderType as RenderAsHTML, then the Extension will be html.

◦       MimeType: [out] The MIME type of the rendered report. the default will be en-us.

◦       Encoding: [out] The encoding used when report server renders the contents of the report

◦       Warnings: [out] An array of Warning objects that describes any warnings that occurred during report processing

StreamIds: [out] the stream id entifiers. These IDs are passed to the RenderStream method. The StreamIds is collection of ids that will identify external resources associated with a given report. Our report contains one such image resource i.e. the image that represents the pie chart in the report

         The SSRSReport library uses the Render2 web service method which processes a report under context and renders it in the specified format. The SSRSReport library will parse the return value of this web service, populate the variables to be returned by reference and return the stream.

 

 

 

 

 

 

8. Show the method RenderStream at

ConsumeReportingService.php file

         We will use SSRSReport::RenderStream to get secondary image stream (stream representing the pie chart in the report) associated with the report.

         SSRSReport:: RenderStream method has the following parameters:,

◦       renderType: An object of type RenderAsCSV, RenderAsEXCEL, RenderAsHTML, RenderAsIMAGE, RenderAsMHTML, RenderAsPDF, RenderAsWORD, RenderAsXML

◦       StreamID: The stream identifier

◦       Encoding: [out] the encoding used to render the content of the report.

◦       MimeType: [out] The MIME type of the stream.

          The SSRSReport library uses the RenderStream web service method which returns a stream that is associated with a rendered report. The SSRSReport library will parse the return value of this web service, populate the variables to be returned by reference and return the stream

          We will save the stream representing the pie chart into the images folder in the current directory with name equal to the stream id. The image url in the report will refer to this folder for associated resources. (we asked report sever to do so by setting RenderAsHTML::StreamRoot member)

          The statement echo $result_html; will display the report in the browser.

 

 

 

 

 

 

 

 

 

 

 

9. Show the method Sort2 at

ConsumeReportingService.php file

         Since we set RenderAsHTML::ReplacementRoot to the url of current page, the url behind the sort link of the report will be as follows:

http://localhost:50002/ConsumeReportingService.php?&ps:SessionID=kgvrtivkkz1atjagmczhopii& ps:OrginalUri=http://localhost/ReportServer_SQLEXPRESS?/AdventureWorks 2008 Sample Reports/Top Score Begin&rs:Command=Sort&&rs:SortId=15iT0&rs:SortDirection=Ascending&rs:ClearSort=TRUE&rc:ReplacementRoot=http//localhost:50002/ ConsumeReportingService.php?&ps:SessionID=kgvrtivkkz1atjagmczhopii&ps:OrginalUri&rc:StreamRoot=./images

The url parameters shown in red color are parameters to the sort2 API.

         Clicking on the sort link will call our page with $_REQUEST global array set.

         We will use SSRSReport::Sort2 to sort the report.

         SSRSReport::Sort2 has the following parameters:

◦       SortItem: The ID of the report item on which to sort

◦       Direction: The direction for the sort

◦       Clear: Indicates whether all other existing sorts should be cleared.

◦       PaginationMode: The mode by which the report is processed. Possible values are:

◦       ReportItem [out]: The ID of the item on the page used for positioning in the viewing area.

◦       ExecutionInfo [out]: An ExecutionInfo2 object containing new execution information.

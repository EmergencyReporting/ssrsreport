Installing IIS 7

In this section you will learn how to install IIS 7 and configure its FastCGI module to host PHP applications. You can either run provided scripts that installs and configures it automatically (Automatic Installation section), or you can perform the same procedure manually to have more control over the installation (Manual Installation section).

Note: The FastCGI module in IIS enables popular application frameworks that support the FastCGI protocol to be hosted on the IIS web server in a high-performance and reliable way. FastCGI provides a high-performance alternative to the Common Gateway Interface (CGI), a standard way of interfacing external applications with Web servers that has been supported as part of the IIS feature-set since the very first release.

1.       Open the Windows Features dialog box. To do this, click Start | Control Panel and double-click Programs and Features (in classic view).

2.       In the Tasks pane, select Turn Windows features on or off.

3.       Enable the default IIS settings. To do this, locate the Internet Information Services node, and select its check box.

Installing PHP for Windows

In this section, you will learn how to install PHP and configure it on your system for hosting PHP applications on IIS 7 using FastCGI. The PHP releases are available in two flavors, a zip package and a msi installer. You can either run a quick procedure by following the steps in the Automatic Installation section (using the PHP msi installer) or you can perform the same procedure manually to have more control over the installation by using Manual Installation section (using the zip package).

 

Manual Installation

1.       Download the latest PHP Non-thread-safe zip package (5.2.8 or above).

Note: It is recommended to use a non-thread safe build of PHP with IIS 7.0 FastCGI. A non-thread safe build of PHP provides significant performance gains over the standard build by not doing any thread-safety checks, which are not necessary, since FastCGI ensures a single threaded execution environment.

2.       Unzip the package under C:\PHP.

3.       Set up a valid configuration file for PHP. To do this, locate the file C:\PHP\php.ini-recommended and rename it to php.ini.

Note: You have also another ini file available to use called php.ini-dist, but the php.ini-recommended has been optimized for better performance and security.

4.       Grant to the php.ini file Read permissions to Everyone. To do this, right-click the file and select Properties. Then in the Security tab select Edit, add the Everyone user and apply the changes.

5.       Modify the Windows Path environment variable to include the PHP installation folder. To do this, right-click Computer | Properties, in the Tasks pane select Advanced system settings and click Environment Variables. In the System Variables section, double-click the Path variable, and append the path C:\PHP (using a semi-colon to separate it).

6.       Grant to the C:\PHP folder Read & Execute permissions to IIS_IUSRS. To do this, right-click the folder and select Properties. Then in the Security tab select Edit, add the IIS_IUSRS user and apply the changes.

7.       Edit the pws-php5cgi.reg file (located in C:\PHP) to reflect the location of your php-cgi.exe. The file should look like the following:

pws-php5cgi.reg

REGEDIT4

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\w3svc\parameters\Script Map]

".php"="C:\\PHP\\php-cgi.exe"

8.       Save the pws-php5cgi.reg file, and execute it to reflect the changes in the registry; you can do this by double-clicking it.

9.       Enable FastCGI in IIS 7 for PHP. To do this, follow these steps:

a.       Browse to the C:\Windows\System32\inetsrv\config folder.

b.      Open the applicationHost.config file.

c.       Find the <fastCgi /> element, and replace it by the following content:

applicationHost.config

<fastCgi>

<application fullPath="c:\PHP\php-cgi.exe" />

</fastCgi>

 

10.   Modify the PHP configuration file (php.ini located under C:\PHP) to set some recommended settings. Uncomment and modify the configuration values in the Paths and Directories section as follows:

php.ini

fastcgi.impersonate=1

cgi.fix_pathinfo=1

cgi.force_redirect=0

Note: FastCGI under IIS supports the ability to impersonate security tokens of the calling client. Setting fastcgi.impersonate = 1 allows IIS to define the security context that the request runs under.

cgi.fix_pathinfo provides *real* PATH_INFO/PATH_TRANSLATED support for CGI. PHP's previous behavior was to set PATH_TRANSLATED to SCRIPT_FILENAME, and to not care what PATH_INFO is. For more information on PATH_INFO, see the cgi specs. Setting this to 1 will cause PHP CGI to fix its paths to conform to the spec.

cgi.force_redirect is necessary to provide security running PHP as a CGI under most web servers. However this is not required if running under IIS.

For more information and recommended practices about PHP on IIS 7 see Using FastCGI to Host PHP Applications on IIS 7.0.

11.   Save and close php.ini. Then restart IIS by executing the iisreset command in a command prompt.

Note: For more info, see install.txt located in the PHP root directory.

 

 

Adding the Handler Mapping for PHP Requests in IIS7

In order for IIS 7.0 to host PHP applications, it is necessary to add a handler mapping that tells IIS to pass all PHP specific requests to the PHP application framework via FastCGI protocol. You can either run a provided script that installs and configures it automatically (Automatic Installation section), or you can perform the same procedure manually to have more control over the installation (Manual Installation section).

Manual Installation

1.       Open IIS Manager from Start | All Programs | Administrative Tools.

Note: If the IIS Manager is not installed, go to Start | Control Panel | Programs and Features | Turn Windows features on or off and select the IIS Managment Console check box located under the Internet Information Services | Web Managment Tools node.

 

2.       At the server level, open Handler Mappings feature ( ).

Figure 9

Opening Handler Mappings in IIS7 at Server Level

Note: The PHP handler will be registered at the server level; this means that IIS7 will let FastCGI work on every site in the server. In a more realistic scenario, you may avoid enabling this at the server level for security reasons; instead, consider registering it separately on each PHP site.

3.       In the Actions pane, select Add Module Mapping, and fill the fields as specified in the following figure. Then click OK.

Figure 10

Registering the PHP Handler via IIS Manager

 

4.       A dialog box will be displayed informing that you are about to create a FastCGI application for the given executable. Click Yes.

Enabling SOAP Extension for PHP

In this section you will learn how to enable SPAP extension for PHP which allow you to access Web-Services from your PHP Web applications.

Manual Installation

1.       Modify the PHP configuration file (php.ini located under C:\PHP) to enable SOAP extension, for this search for extension=php_soap.dll by you will find it under Dynamic Extensions; remove the semi-colon (;) in front of it.

php.ini

extension=php_soap.dll

 

2.       Save and close php.ini. Then restart IIS by executing the iisreset command in a command prompt.

Note: To verify that you correctly enabled SOAP extension, create a php script named phpinfo.php in the root of IIS 7 Default Web Site (C:\inetpub\wwwroot in most cases). Then edit the file and copy the following code: <?PHP phpinfo(); ?>. Finally, open a Web browser and browse to http://localhost/phpinfo.php. You should see SOAP extension listed.

Setting up SSRSReport Demo Web Site

After all the prerequisites are installed, the Configuration Wizard runs deployment and configuration scripts will to create Web site in IIS for the SSRSReport demo.

Web site is configured to use an isolated custom application pool running under the NETWORK SERVICE account. Then the database authentication is performed using integrated security with the application pool identity.

For this environment configuration a login for the NETWORK SERVICE user is created into your SQL Server instance, and then the NETWORKS SERVICE user is granted permissions to access the AdventureWorks database (the NETWORK SERVICE user is mapped with the login created).

These settings follow some best practices for configuring FastCGI and PHP which include:

         Use one application pool per web site

         Use a dedicated user account as an identity for the application pool

         Grant minimum set of NTFS permissions to the Web Site physical path

         Configure anonymous authentication in IIS to use the application pool identity

         Ensure that FastCGI impersonation is enabled in the php.ini file (fastcgi.impersonate=1, done during the SQL Server PHP Driver prerequisite installation)

 

Note: For more information and recommended practices about PHP on IIS 7 see Using FastCGI to Host PHP Applications on IIS 7.0.

In this section you will learn how to setup both Web sites following the recommended practices. You can either run provided scripts that setup everything automatically (Automatic Installation section), or you can perform the same procedure manually to have more control over the installation and learn the details (Manual Installation section).

Task 2 - Creating the SSRSReport Application Pool

An Application Pool can contain one or more applications and allows you to configure a level of isolation between different Web applications even if they are being hosted on a common server. For example, if you want to isolate all the Web applications running in the same computer, you can do this by creating a separate application pool for every Web application and placing them in their corresponding application pool. Because each application pool runs in its own worker process, errors in one application pool will not affect the applications running in other application pools.

In this task you will create an application pool for the training kit Web sites. This configuration follows the general security isolation recommendations for IIS 7.0.

1.       Open IIS Manager from Start | All Programs | Administrative Tools.

2.       Expand the server node and click the Application Pools node ( ).

3.       In the Actions pane click Add Application Pool ( ).

4.       Type SSRSReportDemoAppPool as the Name and click OK.

Introduction

This document provides setup documentation for configuring Reporting Service on your machine and deploying adventure work sample report into the reporting service.

Setup and Configuration

The setup and configuration for this demo involves the following tasks:

         Task 1 Configuring Report Server

         Task 2 Configuring Basic Authentication in Reporting Services

         Task 3 Deploying AdventureWorks Sample Reports

         Task 4 Browsing the Reports

 

 

Task 1 Configuring the Report Server

1.       Open Reporting Services Configuration Manager from Start | All Programs | Microsoft SQL Server 2008 | Configuration Tools.

Note: The Reporting Services Configuration tool is used to configure a Reporting Services installation. You can configure a local or remote report server instance as long as you are a member of the local administrator group on the report server computer.

 

2.       In the Connect to a report server instance dialog box, make sure the local SQLEXPRESS instance is selected and click Connect.

Figure 4

Reporting Services Configuration Manager connection

 

3.       Select the Service Account page ( ) from the left pane.

4.       On the Service Account page, select the Use Built-in Account option and select the Network Service account.

Figure 5

Configure Service Account

 

5.       Click Apply.

6.       When prompted to back up the symmetric key, type a file name (for example, backupKey.snk) and a location for the symmetric key backup file (for example, C:\), then type a password to lock and unlock the file, and finally click OK.

7.       The SQL Server Connection dialog box will be displayed. Select Current User - Integrated Security for the Credentials Type and click OK.

Figure 6

Connection Account

 

8.       On the Web Service URL ( ) page, make sure that the Virtual Directory is set to ReportServer_SQLEXPRESS and that the TCP Port is set to 80.

Note: The virtual directory name identifies which application receives the request. Because an IP address and port can be shared by multiple applications, the virtual directory name specifies which application receives the request.

Figure 7

Web Service Configuration

 

9.       On the Database page( ), make sure that:

a.       SQL Server Name points to the local SQLExpress instance

b.      Database Name is set to ReportServer$SQLEXPRESS

c.       Report Server Mode is set to Native.

Figure 8

Database Configuration

 

10.   If the Database Configuration values are not set as above, follow these steps:

a.       Click Change Database to open the Report Server Database Configuration Wizard.

b.      Click Create a new report server database and click Next.

c.       In the Database Server page, set the Server Name to point the local SQLExpress and select Current User -Integrated Security for the Authentication Type.

Figure 9

Change Database Server Configuration

 

d.      Click Next.

e.      In the Database page, type ReportServer as the Database Name and click Native Mode for the Report Server Mode.

 

Figure 10

Change Database Configuration

 

f.        Click Next to continue.

g.       In the Credentials page, select Service Credentials for the Authentication Type.

Figure 11

Change Database Configuration

 

h.      Click Next, and finally Finish to complete the wizard.

11.   On the Report Manager URL page ( ), make sure that the Virtual Directory is set to Reports_SQLEXPRESS.

Note: Use the Report Manager URL page to configure or modify the URL used to access Report Manager. By default, the Report Manager URL inherits the prefix, IP address, and port of the Report Server Web service URL. This is because Report Manager provides front-end access to the Web service that runs within the same Report Server service.

12.   Open a Web browser with Administrator privileges. and browse to the SQL Server Report Manager at http://localhost/Reports_SQLEXPRESS

14.   In Group or user name text box add PHPDemoUser (PHPdemoUser must be added to the list of logins using SQL Server Management studio: connect to the local SQLExpress DB using .\sqlexpress. In the logins tab, right click and select New Login. Add ad the PHPDemo User or another user that you can use for the samples)

15.   In the Role column select Content Manager and click OK.

16.   Repeat steps 14 and 15 but specify your personal user account.

 

Task 2 Configuring Basic Authentication in Reporting Services

1.       Modify the rsreportserver.config configuration file located under the Report Server installation folder (C:\Program Files\Microsoft SQL Server \MSRS10.SQLEXPRESS\Reporting Services\ReportServer in most common scenarios). You might need to open the text editor with Administrator privileges.

2.       Find the <Authentication> element located under the <Configuration> element and add the basic authentication type (shown in bold) under the <AuthenticationTypes> element.

XML

<AuthenticationTypes>

<RSWindowsBasic/>

</AuthenticationTypes>

Note: If other AuthenticationTypes entries are present, do not delete them. Just add the <RSWindowsBasic/> element at the end.

Figure 15

rsreportserver.config sample configuration

Note: In previous versions of Reporting Services, authentication support was provided by IIS. In this release, IIS is no longer used. Reporting Services handles all authentication requests internally. Basic authentication (RSWindowsBasic) is defined in the HTTP protocol and can only be used to authenticate HTTP requests to the report server. Credentials are passed in the HTTP request in base64 encoding.

 

3.       Save and close the rsreportserver.config file.

4.       Start SQL Server Configuration Manager from Start | All Programs | Microsoft SQL Server 2008 | Configuration Tools | SQL Server Configuration Manager.

5.       Select SQL Server Services, right-click SQL Server Reporting Services, and click Restart.

 

Task 3 Deploying the AdventureWorks Sample Reports

1.       Open Business Intelligence Development Studio from Start | All Programs | Microsoft SQL Server 2008 with administrative privileges.

2.       Click the File menu, point to Open and then click Project/Solution.

3.       Browse to the AdventureWorks Sample Reports directory, select the file AdventureWorks 2008 Sample Reports.sln and click Open.

Figure 16

AdventureWorks2008 Sample Reports in Business intelligence Development Studio

 

4.       On the Visual Studio toolbar, select Release in the Solution Configurations (the default configuration is Debug).

Figure 17

Release Configuration

 

5.       On the Project menu, click Properties.

6.       On the General tab, configure the General properties as follows:

a.       Set OverwriteDataSources to True.

b.      Set TargetServerURL to: http://localhost/reportserver_SQLExpress

Figure 18

Project Configuration

 

7.       Click OK.

8.       In Solution Explorer, under the Shared Data Sources folder, double-click AdventureWorks.rds to open the Adventure Works Data Source Properties.

9.       Under the General page, modify the Connection String to use the following value:

Connection String

Data Source=localhost\SQLExpress; Initial Catalog=AdventureWorks

Figure 19

Data Source Configuration

 

10.   Click OK.

11.   On the Build menu, click Deploy AdventureWorks 2008 Sample Reports.

Note: When you deploy the solution, the reports are published to the report server.

 

Task 4 Browsing the Reports

1.       Open a Web browser with administrator privileges.

Note: It is important to run the Web browser as administrator otherwise, you won't be able to browse the reports.

 

2.       Browse Report Manager at http://localhost/Reports_SQLEXPRESS/Pages/Folder.aspx?ItemPath=%2fAdventureWorks+2008+Sample+Reports&ViewMode=List. You should see all AdventureWorks2008 Sample Reports.

Note: Report Manager is a Web-based tool that includes features for viewing and managing reports. It is part of a report server installation. To open Report Manager, type the Report Manager URL in a browser window. You can use any supported Web browser to view a report through a direct connection to a report server. Every report has a URL address on a report server. You can use that address to open it in a new browser window independently.
